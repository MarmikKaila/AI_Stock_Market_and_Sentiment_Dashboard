================================================================================
                          1. PROJECT OVERVIEW
================================================================================

This is a FULL-STACK web application that:
- Displays stock market information (price, P/E ratio, P/B ratio, EPS, etc.)
- Fetches real-time news about stocks
- Analyzes news sentiment using AI (Google Gemini API)
- Provides AI-powered BUY/HOLD/SELL recommendations
- Stores data in MongoDB for caching

TECHNOLOGIES USED:
------------------
Frontend: React.js + Vite + Tailwind CSS
Backend: Node.js + Express.js
Database: MongoDB (via Mongoose ODM)
AI Services: Google Gemini API
External APIs: Alpha Vantage (Stock data), NewsAPI (News articles)

4.1 server.js - MAIN ENTRY POINT
--------------------------------------------------------------------------------

LOCATION: server/server.js

PURPOSE: 
This is the starting point of the backend application. When you run 
"node server.js" or "npm start", this file executes first.

WHAT IT DOES:
1. Loads environment variables from .env file (API keys, database URL)
2. Creates an Express.js application
3. Connects to MongoDB database
4. Sets up middleware (CORS, JSON parsing)
5. Imports and mounts all route files
6. Starts the server on port 5000

CODE BREAKDOWN:
---------------
require('dotenv').config();   // Load .env file variables into process.env
const express = require('express');
const cors = require('cors');
const connectDB = require('./config/db');

const app = express();       // Create Express application

connectDB();                 // Connect to MongoDB

// MIDDLEWARE SETUP:
app.use(cors());             // Allow frontend (different port) to make requests
app.use(express.json());     // Parse JSON in request body
app.use(express.urlencoded({ extended: true })); // Parse URL-encoded data

// ROUTE MOUNTING:
app.use('/api/stocks', stockRoutes);    // All /api/stocks/* go to stockRoutes
app.use('/api/opinion', opinionRoutes); // All /api/opinion/* go to opinionRoutes
app.use('/api/ai', aiRoutes);           // All /api/ai/* go to aiRoutes

// Health check endpoint - useful to verify server is running
app.get('/api/health', (req, res) => {
  res.json({ status: 'OK', message: 'Server is running' });
});

// ERROR HANDLING - catches any unhandled errors
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).json({ error: 'Something went wrong!' });
});

// START SERVER
const PORT = process.env.PORT || 5000;
app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});

CONNECTION TO OTHER FILES:
- Imports connectDB from config/db.js
- Imports stockRoutes from routes/stockRoutes.js
- Imports opinionRoutes from routes/opinionRoutes.js
- Imports aiRoutes from routes/aiRoutes.js

4.2 config/db.js - DATABASE CONNECTION
--------------------------------------------------------------------------------

LOCATION: server/config/db.js

PURPOSE:
Establishes connection to MongoDB database using Mongoose.

WHAT IT DOES:
1. Gets MongoDB connection string from environment variables
2. Uses Mongoose to connect to the database
3. Handles connection errors gracefully

CODE BREAKDOWN:
---------------
const mongoose = require("mongoose");

const connectDB = async () => {
  try {
    const uri = process.env.MONGO_URI;  // Get DB URL from .env file
    
    if (!uri) {
      console.warn("MONGO_URI not set — skipping DB connection.");
      return;  // App can still run without DB (for development)
    }
    
    await mongoose.connect(uri, {
      useNewUrlParser: true,       // Use new URL parser
      useUnifiedTopology: true,    // Use new topology engine
    });
    
    console.log("MongoDB connected");
  } catch (err) {
    console.error("MongoDB connection error:", err);
  }
};

module.exports = connectDB;

IMPORTANT NOTES:
- MONGO_URI should be in .env file like: MONGO_URI=mongodb://localhost:27017/stockdb
- Can also use MongoDB Atlas cloud URL
- If no URI provided, app continues without database (useful for testing)

4.3 models/Stock.js - DATA MODEL (SCHEMA)
--------------------------------------------------------------------------------

LOCATION: server/models/Stock.js

PURPOSE:
Defines the structure of stock data stored in MongoDB.

WHAT IT DOES:
1. Defines NewsItem schema (sub-document for news articles)
2. Defines Stock schema (main document)
3. Creates and exports Mongoose model

CODE BREAKDOWN:
---------------
const mongoose = require("mongoose");

// SUB-SCHEMA: Each news article has these fields
const NewsItemSchema = new mongoose.Schema({
  title: String,              // News headline
  url: String,                // Link to full article
  publishedAt: Date,          // When article was published
  sentiment: String,          // "Positive", "Negative", or "Neutral"
});

// MAIN SCHEMA: Each stock document has these fields
const StockSchema = new mongoose.Schema({
  symbol: {                   // Stock ticker symbol (e.g., "AAPL")
    type: String, 
    required: true, 
    index: true              // Indexed for fast lookups
  },
  name: String,              // Company name (e.g., "Apple Inc")
  price: Number,             // Current stock price
  peRatio: Number,           // Price-to-Earnings ratio
  pbRatio: Number,           // Price-to-Book ratio
  eps: Number,               // Earnings Per Share
  marketCap: Number,         // Market Capitalization
  lastFetchedAt: Date,       // When data was last updated
  news: [NewsItemSchema],    // Array of news articles
  sentimentScore: Number,    // Overall sentiment (-1 to +1)
  recommendation: String,    // AI recommendation: "BUY"/"HOLD"/"SELL"
}, { timestamps: true });    // Adds createdAt and updatedAt fields

// Create model from schema and export
module.exports = mongoose.model("Stock", StockSchema);

HOW THIS CONNECTS:
- Used by stockController.js to save/retrieve stock data
- Used by opinionController.js to get stock data for AI analysis
- MongoDB stores this as a "stocks" collection

4.5 controllers/ - REQUEST HANDLERS
--------------------------------------------------------------------------------

Controllers contain the logic that runs when an API endpoint is called.

--- stockController.js ---

LOCATION: server/controllers/stockController.js

PURPOSE: Handles requests for stock data

WHAT IT DOES:
1. Checks if stock data is cached in database (less than 30 min old)
2. If cached, returns cached data immediately
3. If not cached, fetches fresh data from external APIs
4. Calculates sentiment score from news articles
5. Saves data to database for future caching
6. Returns data to frontend

CODE BREAKDOWN:
---------------
const Stock = require("../models/Stock");
const { fetchFundamentalsAlpha, fetchLatestPrice } = require("../services/stockService");
const { fetchNews } = require("../services/newsService");

async function getStock(req, res) {
  try {
    const symbol = req.params.symbol.toUpperCase();  // Get symbol from URL

    // CHECK CACHE: Is data in database and recent?
    const cache = await Stock.findOne({ symbol });
    const THIRTY_MIN = 1000 * 60 * 30;  // 30 minutes in milliseconds
    
    if (cache && (Date.now() - new Date(cache.lastFetchedAt).getTime()) < THIRTY_MIN) {
      return res.json({ fromCache: true, data: cache });
    }

    // NO CACHE: Fetch fresh data
    const fundamentals = await fetchFundamentalsAlpha(symbol);  // Get PE, PB, EPS, etc.
    const price = await fetchLatestPrice(symbol);               // Get current price
    fundamentals.price = price;

    const news = await fetchNews(symbol, fundamentals.name);    // Get news articles
    const sentimentScore = computeSentimentScore(news);         // Calculate sentiment

    // SAVE TO DATABASE (upsert = update if exists, insert if not)
    const doc = await Stock.findOneAndUpdate(
      { symbol },
      {
        symbol,
        name: fundamentals.name,
        price: fundamentals.price,
        peRatio: fundamentals.peRatio,
        pbRatio: fundamentals.pbRatio,
        eps: fundamentals.eps,
        marketCap: fundamentals.marketCap,
        news,
        sentimentScore,
        lastFetchedAt: new Date(),
      },
      { upsert: true, new: true }
    );

    return res.json({ fromCache: false, data: doc });
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
}

// HELPER: Calculate sentiment score from news items
function computeSentimentScore(newsItems) {
  if (!newsItems || newsItems.length === 0) return 0;
  
  let score = 0;
  newsItems.forEach(n => {
    if (n.sentiment === "Positive") score += 1;
    else if (n.sentiment === "Negative") score -= 1;
    // Neutral = 0, no change
  });
  
  return score / newsItems.length;  // Normalize to -1 to +1 range
}


--- opinionController.js ---

LOCATION: server/controllers/opinionController.js

PURPOSE: Gets AI recommendation for a stock

WHAT IT DOES:
1. Looks up stock data from database
2. Returns error if stock not found (user must call /api/stocks first)
3. Extracts fundamentals and news summary
4. Calls AI service to get recommendation
5. Saves recommendation to database
6. Returns recommendation to frontend

CODE BREAKDOWN:
---------------
const Stock = require("../models/Stock");
const { getRecommendation } = require("../services/aiService");

async function getOpinion(req, res) {
  try {
    const symbol = req.params.symbol.toUpperCase();

    // GET STOCK FROM DATABASE
    let stock = await Stock.findOne({ symbol });
    
    if (!stock) {
      return res.status(404).json({ 
        error: "Stock not found. Call /api/stocks/:symbol first." 
      });
    }

    // PREPARE DATA FOR AI
    const fundamentals = {
      peRatio: stock.peRatio,
      pbRatio: stock.pbRatio,
      eps: stock.eps,
      price: stock.price,
      marketCap: stock.marketCap,
    };

    // Get first 3 news headlines as summary
    const newsSummary = (stock.news || [])
      .slice(0, 3)
      .map(n => n.title)
      .join(" | ");
    
    const sentimentScore = stock.sentimentScore;

    // CALL AI SERVICE
    const aiResult = await getRecommendation({ 
      fundamentals, 
      sentimentScore, 
      newsSummary 
    });

    // SAVE RECOMMENDATION
    stock.recommendation = aiResult;
    await stock.save();

    res.json({ symbol, recommendation: aiResult });
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
}

4.6 services/ - BUSINESS LOGIC & EXTERNAL API CALLS
--------------------------------------------------------------------------------

Services contain the business logic and external API integrations.

--- stockService.js ---

LOCATION: server/services/stockService.js

PURPOSE: Fetches stock data from Alpha Vantage API

FUNCTIONS:

1. fetchFundamentalsAlpha(symbol):
   - Calls Alpha Vantage OVERVIEW endpoint
   - Returns: name, peRatio, pbRatio, eps, marketCap
   - Falls back to mock data if API fails

2. fetchLatestPrice(symbol):
   - Calls Alpha Vantage GLOBAL_QUOTE endpoint
   - Returns: current stock price
   - Falls back to random mock price if API fails

MOCK DATA:
- When ALPHA_VANTAGE_API_KEY is not set or API fails
- Returns realistic-looking fake data for development


--- newsService.js ---

LOCATION: server/services/newsService.js

PURPOSE: Fetches news and analyzes sentiment

WHAT IT DOES:
1. Finds a working Google Gemini model (tries multiple versions)
2. Fetches news from NewsAPI for given stock
3. Analyzes sentiment of each headline using Gemini AI
4. Falls back to keyword-based sentiment if AI fails
5. Returns news array with sentiment labels

KEY FUNCTIONS:

1. findWorkingModel():
   - Tries different Gemini model versions
   - Caches the working model for future calls
   - Returns null if no model works

2. analyzeSentiment(text):
   - Sends headline to Gemini API
   - Prompt: "Analyze sentiment. Respond: Positive/Negative/Neutral"
   - Returns sentiment label

3. analyzeSentimentFallback(text):
   - Uses keyword matching when Gemini fails
   - Positive words: 'strong', 'growth', 'profit', etc.
   - Negative words: 'loss', 'decline', 'fail', etc.

4. fetchNews(symbol, companyName):
   - Calls NewsAPI with company name as search query
   - Processes each article through sentiment analysis
   - Returns array of news objects with sentiment

5. getMockNews():
   - Returns fake news articles for development
   - Used when NEWSAPI_KEY is not configured


--- aiService.js ---

LOCATION: server/services/aiService.js

PURPOSE: Provides AI-powered stock analysis and recommendations

WHAT IT DOES:
1. Uses Google Gemini API for AI capabilities
2. Generates BUY/HOLD/SELL recommendations
3. Falls back to rule-based logic if AI fails
4. Can summarize news articles

KEY FUNCTIONS:

1. callGeminiAPI(messages, options):
   - Generic function to call Gemini API
   - Handles errors and retries
   - Returns AI response text

2. generateDetailedRecommendation({ fundamentals, sentimentScore, newsSummary }):
   - Main recommendation function
   - Sends prompt with all stock data to Gemini
   - Asks for single word: BUY, HOLD, or SELL
   - Falls back to getSimpleRecommendation if AI fails

3. getSimpleRecommendation(fundamentals, sentimentScore):
   - Rule-based fallback recommendation
   - Scores based on:
     * PE Ratio (lower = better)
     * PB Ratio (lower = better)
     * EPS (higher = better)
     * Sentiment (positive = better)
   - Score >= 3: BUY
   - Score <= -2: SELL
   - Otherwise: HOLD

4. summarizeNews(article):
   - Summarizes news article in 2-3 sentences
   - Falls back to truncating text if AI fails

5.5 services/stockService.js - API COMMUNICATION
--------------------------------------------------------------------------------

LOCATION: client/src/services/stockService.js

PURPOSE: 
Makes HTTP requests from frontend to backend API.
This is how the frontend communicates with the backend.

CODE BREAKDOWN:
---------------
const BACKEND_URL = 'http://localhost:5000';  // Backend server address

export const getStockDetails = async (symbol) => {
  try {
    // Fetch BOTH endpoints in parallel for efficiency
    const [stockResponse, opinionResponse] = await Promise.all([
      fetch(`${BACKEND_URL}/api/stocks/${symbol}`),
      fetch(`${BACKEND_URL}/api/opinion/${symbol}`),
    ]);

    // Check if responses are OK
    if (!stockResponse.ok || !opinionResponse.ok) {
      throw new Error('Network response was not ok.');
    }

    // Parse JSON responses
    const stockResult = await stockResponse.json();
    const opinionResult = await opinionResponse.json();

    // Combine and return
    return {
      stock: stockResult.data,    // Stock fundamentals, news, etc.
      opinion: opinionResult,     // AI recommendation
    };
  } catch (error) {
    console.error(`Failed to fetch details for ${symbol}:`, error);
    throw error;  // Let Dashboard handle the error
  }
};

KEY POINTS:
- Uses browser's native fetch() API
- Calls two endpoints simultaneously (Promise.all)
- Combines results into single object
- throws errors for Dashboard to catch

================================================================================
                    6. HOW FRONTEND CONNECTS TO BACKEND
================================================================================

CONNECTION MECHANISM: HTTP Requests via fetch() API

FLOW:
-----
1. User enters stock symbol (e.g., "AAPL") in Navbar search
2. Dashboard.jsx receives symbol via handleSearch callback
3. Dashboard updates currentSymbol state
4. useEffect triggers, calling getStockDetails(symbol)
5. stockService.js makes HTTP requests:
   - GET http://localhost:5000/api/stocks/AAPL
   - GET http://localhost:5000/api/opinion/AAPL
6. Backend processes requests and returns JSON
7. stockService.js combines responses
8. Dashboard receives data and updates state
9. React re-renders UI with new data

CORS (Cross-Origin Resource Sharing):
-------------------------------------
- Frontend runs on port 5173 (Vite dev server)
- Backend runs on port 5000
- Without CORS, browser blocks cross-origin requests
- Backend uses cors() middleware to allow frontend requests

IN server.js:
const cors = require('cors');
app.use(cors());  // Allows requests from any origin


================================================================================
                      7. HOW DATABASE IS CONNECTED
================================================================================

DATABASE: MongoDB
ODM LIBRARY: Mongoose

CONNECTION FLOW:
----------------
1. .env file contains: MONGO_URI=mongodb://localhost:27017/stockdb
2. server.js calls: connectDB()
3. config/db.js reads MONGO_URI from process.env
4. Mongoose connects: await mongoose.connect(uri, options)
5. Connection established - Mongoose tracks connection state

DATA STORAGE:
-------------
- Stock.js defines schema (structure of data)
- Controllers use Stock model to interact with DB
- Mongoose translates JavaScript calls to MongoDB queries

CACHING MECHANISM:
------------------
1. User requests stock data
2. Controller checks: Is data in DB and < 30 min old?
3. If YES: Return cached data immediately
4. If NO: Fetch from external APIs, save to DB, return

WHY CACHING?
- External APIs have rate limits
- Reduces API costs
- Faster response times for repeated requests
